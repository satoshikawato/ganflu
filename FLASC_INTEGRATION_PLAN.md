# FLASC Integration Plan

## Goal

Integrate the useful part of `../flasc` into `ganflu` as an optional IAV amino acid variant scanning step.

The first implementation will:

- support IAV only
- be disabled by default
- emit TSV output only
- avoid writing variant annotations back into GenBank
- keep the scanner importable and testable without running `miniprot`

## CLI Design

Add optional mutation scanning to the existing `ganflu` command.

```bash
ganflu \
  -i input.fa \
  -t IAV \
  -o output/sample \
  --scan-mutations strict
```

Proposed options:

```text
--scan-mutations {off,all,variants,strict}
--mutation-db FILE
--mutation-output FILE
--mafft-bin FILE
```

Defaults:

```text
--scan-mutations off
--mutation-db ganflu/db/IAV/mutations.toml
--mutation-output <out_stem>.variants.tsv
--mafft-bin mafft
```

`--scan-mutations off` preserves current behavior.
When mutation scanning is enabled, MAFFT is required.

## Scope

### In Scope

- Copy and normalize the FLASC mutation definition data into `ganflu/db/IAV/`.
- Add reference protein sequences required for mutation coordinate mapping.
- Extract CDS translations from the GenBank file produced by `ganflu`.
- Align query/reference proteins with MAFFT for coordinate mapping.
- Scan known amino acid substitutions and cleavage-site patterns.
- Write `<out_stem>.variants.tsv`.
- Add focused unit tests for the scanner core.

### Out of Scope For Initial Version

- IBV mutation scanning.
- GenBank `/note` mutation annotation.
- A separate `flasc` command bundled inside `ganflu`.
- Full clinical or risk interpretation beyond the notes already present in the mutation DB.

## Data Layout

Proposed package files:

```text
ganflu/db/IAV/mutations.toml
ganflu/db/IAV/prot/mutation_refs.faa
```

`mutations.toml` will be based on `../flasc/mutations.toml`, but normalized where needed:

- convert coordinate strings to integers
- ensure every substitution has a usable reference sequence
- keep `wildtype` as either a string or list, but normalize internally
- keep `type = "cleavage_site"` entries for pattern-based scanning

`mutation_refs.faa` should contain exact reference proteins for mutation numbering, not just the current `IAV_proteome_consensus.faa`, because some FLASC references do not exactly match the GANFLU consensus proteins.

## Reference Assignment

For definitions with `protein_id = ""`, assign default references by gene.

Initial mapping:

```text
PB1 -> PB1
PA  -> PA
M1  -> M1
M2  -> M2
NS  -> NS1
```

Definitions with explicit reference IDs keep those IDs:

```text
PB2 -> YP_308664.1
HA  -> P03437.1
NA  -> P03473.1
```

If a mutation definition has `gene_id = "NS"` and no more specific product, use `NS1` for the initial implementation because the current FLASC `P42S` definition is a single NS-section mammalian-adaptation substitution and matches the common NS1 numbering use case. If later NS2 mutations are added, split the DB key or add an explicit `product_id`.

## Scanner Architecture

Add an importable scanner module, for example:

```text
ganflu/scripts/mutation_scan.py
```

Core API sketch:

```python
scan_translations(
    translations: dict[str, str],
    mutation_db_path: str,
    reference_fasta_path: str,
    mode: str = "strict",
    mafft_bin: str = "mafft",
) -> list[VariantCall]
```

Where `translations` maps normalized gene or product names to amino acid sequences:

```python
{
    "PB2": "...",
    "HA": "...",
    "NA": "...",
    "M2": "...",
}
```

Use a dataclass for output rows:

```python
VariantCall(
    query: str,
    gene: str,
    coordinate: int | None,
    variant: str,
    reference: str,
    residue: str,
    mode: str,
    note: str,
)
```

## Translation Extraction

After `gff3togbk` writes `<out_stem>.gbk`, read the GenBank file and collect CDS translations.

Required normalization:

- `product = "PB2"` -> `PB2`
- `product = "HA"` or `hemagglutinin` -> `HA`
- `product = "NA"` or `neuraminidase` -> `NA`
- `matrix protein 1` -> `M1`
- `matrix protein 2` -> `M2`
- `nonstructural protein 1` -> `NS1`
- `nonstructural protein 2` or `NEP` -> `NS2`

The scanner should not depend on FASTA record IDs generated by `ganflu`; it should scan CDS translations by product/gene identity.

## Coordinate Mapping

### Preferred Behavior

For each query protein and its mutation reference:

1. align query protein and the reference protein with MAFFT
2. map reference residue coordinate to the aligned query residue
3. report the query residue at that reference coordinate

This keeps coordinate mapping behavior closer to the FLASC MSA-oriented model and avoids relying on pairwise alignment heuristics for HA/NA numbering.

### MAFFT Requirement

Mutation scanning requires MAFFT.

Implementation requirements:

- check `mafft` with `shutil.which()` before scanning
- allow an alternate binary path via `--mafft-bin`
- fail with a clear error if MAFFT is missing
- write temporary reference/query FASTA files in the work directory or a managed temporary directory
- parse the MAFFT-aligned FASTA with Biopython
- cache alignments per `(reference_id, query_id)` within a run to avoid repeated work

Packaging requirements:

- add MAFFT to the conda recipe runtime dependencies
- document that `--scan-mutations` requires MAFFT on `PATH`

## Scan Modes

For substitution definitions:

```text
all      output every scanned mutation site
variants output if query residue differs from defined wildtype residue(s)
strict   output only if query residue matches defined mutant residue(s)
```

Rules:

- `coordinate` is normalized to `int`.
- `wildtype` can be a string or list.
- `mutant` can be a string or list.
- gaps and missing coordinates are represented explicitly in TSV.
- ambiguous residues are reported but only count as `strict` if they exactly match the mutant definition.

For cleavage-site definitions:

- use regex matching against the query HA protein sequence
- output a row when the mutant cleavage-site pattern is present
- keep coordinate empty unless a reliable motif coordinate is computed

## TSV Output

Header:

```text
query	gene	coordinate	variant	reference	residue	mode	note
```

Example:

```text
sample_HA	HA	160	T160A	T	A	strict	H3 numbering; mammalian adaptation
```

`query` should identify the scanned CDS, preferably using the GenBank record ID plus normalized gene name.

## Integration Flow

Current flow:

```text
input FASTA
  -> miniprot
  -> GFF3
  -> GenBank
```

New optional flow:

```text
input FASTA
  -> miniprot
  -> GFF3
  -> GenBank
  -> extract CDS translations
  -> scan mutations
  -> variants TSV
```

Implementation points:

- add CLI options in `ganflu/ganflu.py`
- run scanner only when `--scan-mutations` is not `off`
- log mutation DB path, reference FASTA path, scan mode, MAFFT binary, and output TSV path
- keep scanner exceptions fatal by default, because silent variant-scan failure would be misleading

## Tests

Add tests under `tests/`.

Core scanner tests:

- HA sample from FLASC detects `T160A` in `strict` mode.
- PB2 string coordinates such as `"627"` are normalized and do not raise `KeyError`.
- `all`, `variants`, and `strict` produce distinct expected outputs.
- list wildtype definitions such as `["T", "I"]` work.
- `protein_id = ""` entries resolve through default reference assignment.
- `HA.FCS` regex detection works.
- missing MAFFT produces a clear error when mutation scanning is enabled.

Integration-level tests:

- scanner can read a small GenBank fixture with CDS translations.
- `--scan-mutations off` does not create a variants TSV.
- `--scan-mutations strict` creates a TSV at the default path.
- `--mutation-output custom.tsv` writes to the requested path.

Avoid requiring `miniprot` for scanner tests. MAFFT is required for mutation-scanner tests that exercise coordinate mapping.

## Implementation Milestones

1. Add packaged IAV mutation DB and mutation reference FASTA.
2. Implement mutation DB loader and normalization.
3. Implement reference/query coordinate mapping with MAFFT.
4. Implement substitution scanning modes.
5. Implement HA cleavage-site regex scanning.
6. Implement GenBank CDS translation extraction.
7. Wire scanner into `ganflu.py` behind `--scan-mutations`.
8. Add TSV writer.
9. Add tests and README usage notes.
10. Add MAFFT dependency documentation and recipe updates.

## Known Risks

- HA and NA numbering are sensitive to reference choice. The reference FASTA used for scanning must contain exact FLASC numbering references.
- The existing GANFLU consensus proteins are not guaranteed to match the FLASC reference proteins.
- `NS` definitions are currently ambiguous; initial mapping to `NS1` should be documented in the DB.
- Cleavage-site scanning is pattern-based and should not be mixed with coordinate-based substitution logic.
- MAFFT is an external runtime dependency for mutation scanning, so installation and error messages must be explicit.

## Acceptance Criteria

- Existing `ganflu` behavior is unchanged when mutation scanning is off.
- `ganflu --scan-mutations strict -t IAV ...` writes a TSV with deterministic headers.
- Scanner unit tests pass without `miniprot`; coordinate-mapping tests run with MAFFT.
- FLASC HA sample behavior is reproduced for the known `T160A` strict-mode example.
- PB2 definitions with string coordinates no longer fail.
- Package builds include the mutation DB and reference FASTA.
